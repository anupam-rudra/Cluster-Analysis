# Customer Segmentation using Clustering and XGBoost

## Project Overview
Customer segmentation is an important task in retail analytics that helps businesses understand purchasing behavior and design targeted marketing strategies.

This project performs **customer segmentation using clustering techniques** and then builds a **machine learning model to predict cluster membership for new customers**.

The workflow combines **unsupervised learning (clustering)** and **supervised learning (classification)**.

---

## Dataset
The project uses the **Wholesale Customers Dataset** from the UCI Machine Learning Repository.

The dataset contains annual spending amounts for wholesale customers across different product categories.

### Features

| Feature | Description |
|---|---|
| Fresh | Annual spending on fresh products |
| Milk | Annual spending on milk products |
| Grocery | Annual spending on grocery items |
| Frozen | Annual spending on frozen products |
| Detergents_Paper | Annual spending on detergents and paper products |
| Delicatessen | Annual spending on delicatessen items |
| Channel | Customer channel (Hotel/Restaurant/Cafe or Retail) |
| Region | Customer region |

Number of observations: **440 customers**

---

## Project Objectives

1. Explore and understand customer purchasing patterns.
2. Identify customer segments using clustering.
3. Evaluate cluster quality.
4. Train a supervised learning model to predict cluster membership.
5. Analyze and interpret customer segments.

---

## Methodology

### 1. Data Preprocessing

Steps performed:

- Missing value treatment
- Outlier detection using the IQR method
- Feature scaling using StandardScaler
- Train-test split

Categorical variables (`Channel`, `Region`) were excluded from clustering since clustering algorithms work best with continuous variables.

---

### 2. Determining Optimal Clusters

The **Elbow Method** was used to determine the optimal number of clusters.

The method analyzes **within-cluster sum of squares (WCSS)** across different cluster counts and identifies the point where adding more clusters yields diminishing improvements.

---

### 3. Clustering Model

Customer segmentation was performed using the **K-Means Clustering** algorithm.

K-Means groups customers based on similarity in purchasing behavior.

Cluster quality was evaluated using the **Silhouette Score**, which measures how well-separated clusters are.

---

### 4. Cluster Visualization

To visualize high-dimensional customer data, **Principal Component Analysis (PCA)** was applied to reduce features to two dimensions.

This allows clusters to be visualized in a 2D space.

---

### 5. Cluster Prediction Model

After clustering the training data, cluster labels were used as **pseudo-labels** for supervised learning.

An **XGBoost classifier** was trained to predict cluster membership.

This allows fast assignment of new customers to existing segments.

## Model Evaluation

The supervised model was evaluated by comparing predicted cluster labels with the cluster assignments generated by K-Means.

Evaluation metrics include:

- Accuracy score
- Confusion matrix
- Classification report

---

## Cluster Interpretation

After clustering, average spending patterns were analyzed for each cluster.

This helps interpret customer segments such as:

- High grocery spending customers
- Fresh product buyers
- Balanced purchasing customers

These insights can be used for targeted marketing and customer relationship management.

---

## Key Insights

The analysis reveals meaningful customer segments based on purchasing behavior.

Different clusters represent distinct purchasing profiles, which could help businesses:

- Design personalized marketing campaigns
- Optimize inventory management
- Improve customer retention strategies
